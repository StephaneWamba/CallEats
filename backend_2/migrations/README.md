-- Migration: 005 - README
-- This directory contains Supabase migration files for the Restaurant Voice Assistant backend.
--
-- Migration Order:
--   001_initial_schema.sql - Core schema (tables, indexes, functions, RLS)
--   002_phone_mappings.sql - Phone number to restaurant mapping
--   003_users_table.sql - User authentication table
--   004_delivery_zones_geometry.sql - PostGIS spatial support
--
-- Usage:
--   These migrations should be run in order using Supabase CLI:
--   $ supabase db reset (runs all migrations)
--   $ supabase migration up (applies pending migrations)
--
--   Or via Supabase Dashboard:
--   1. Go to Database > Migrations
--   2. Click "New Migration"
--   3. Copy and paste migration SQL
--   4. Run migration
--
-- Notes:
--   - Migrations are idempotent (use IF NOT EXISTS)
--   - PostGIS extension requires enabling in Supabase Dashboard if not available
--   - RLS policies ensure service_role has full access for backend operations
--   - Public read access is granted for frontend queries
